# Fluxo Desktop

Fluxo desktop/web client based on React, Mantine, TanStack Query, TanStack Router, Axios and Tailwind CSS v4.

## Architecture

- `src/app`: app bootstrap, providers, shell, router and global styles
- `src/shared`: reusable utilities, API client, shared UI and shared types
- `src/modules`: isolated business modules generated with the scaffold command
- `src/routes`: file-based TanStack Router route definitions

## Auth flow for v1

Fluxo v1 uses the enterprise access key generated by the backend after subscription approval.

- the desktop app stores the key locally
- Axios sends it automatically through `x-module-key`
- the app validates it against `/api/access/me`
- the UI adapts itself to the company, plan and enabled modules returned by the backend

## Generation automatique des routes

Le projet utilise TanStack Router CLI pour générer automatiquement l'arbre des routes à partir du dossier `src/routes`.

```bash
pnpm run router
```

Le fichier généré est `src/routeTree.gen.ts`.
Ne pas modifier ce fichier à la main.

## Scripts utiles

- `dev` : démarre le serveur de développement après génération des routes
- `build` : build de production après génération des routes
- `preview` : prévisualisation du build
- `router` : génère l'arbre des routes
- `generate:module` : crée la structure de base d'un module métier
- `check` : vérifie TypeScript
