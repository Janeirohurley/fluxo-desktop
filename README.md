# Fluxo Desktop

Fluxo multi-tenant ERP desktop client built with Tauri, React, TanStack Router, TanStack Query, Axios and Tailwind CSS v4.

## Architecture

- `src/app`: bootstrap, providers, shell, router and global styles
- `src/shared`: reusable utilities, API client, shared UI and shared types
- `src/modules`: isolated business modules generated with the scaffold command
- `src/routes`: file-based TanStack Router route definitions

## Auth flow for v1

Fluxo v1 uses the enterprise access key generated by the backend after subscription approval.

- the desktop app stores the key locally
- Axios sends it automatically through `x-module-key`
- the app validates it against `/api/access/me`
- the UI adapts itself to the company, plan and enabled modules returned by the backend

## Generation automatique des routes

Le projet utilise TanStack Router CLI pour generer automatiquement l'arbre des routes a partir du dossier `src/routes`.

```bash
pnpm run router
```

Le fichier genere est `src/routeTree.gen.ts`.
Ne pas modifier ce fichier a la main.

## Scripts utiles

- `dev` : demarre le serveur de developpement apres generation des routes
- `build` : build de production apres generation des routes
- `preview` : previsualisation du build
- `router` : genere l'arbre des routes
- `generate:module` : cree la structure de base d'un module metier
- `check` : verifie TypeScript
- `build:desktop` : construit l'application Tauri desktop
- `sync:tauri-version` : aligne `src-tauri/Cargo.toml` et `src-tauri/tauri.conf.json` sur la version de `package.json`
- `release` : met a jour la version, le changelog et cree un tag Git
- `release:push` : pousse `main` et les tags de release

## Versionning et release

Le projet utilise `standard-version` pour gerer le versionning semantique et le changelog.
`package.json` est la source de verite pour la version. Pendant `pnpm run release`, la version Tauri est automatiquement synchronisee dans `src-tauri/Cargo.toml` et `src-tauri/tauri.conf.json` avant le commit et le tag Git.

Preparer une release :

```bash
pnpm run release
```

Pousser la release :

```bash
pnpm run release:push
```

Le push du tag `vX.Y.Z` declenche ensuite le workflow GitHub Actions de release.

La release GitHub :
- publie les notes depuis `CHANGELOG.md`
- construit le frontend web
- construit l'application Tauri Windows
- attache les bundles desktop a la release correspondant au tag

## Notes de release

Les notes GitHub Release sont extraites automatiquement depuis `CHANGELOG.md` par :

```bash
node scripts/extract-release-notes.js vX.Y.Z release-notes.md
```
